# 游戏概念 · 1v1 核心版

> 状态:Draft v0.6 · 2026-06-27(底数×倍率计分模型 + 调参面板)
> 范围:1v1 单机/对战核心循环。3v3、题库、联机、商业化均**不在本版**。
> 本版方向:**底数 × 倍率(chips × mult)计分**(学 Balatro),强化分 +底数 / +倍率 / **×倍率**,
> ×倍率连乘 = 指数爆炸,这才是真正上瘾的来源。新增**调参面板**(数值边玩边拨)。
> 这份文档配套可玩原型:`prototypes/core-loop/index.html`,设计与代码同步演进。

---

## 一、一句话概念

**实时 Roguelike 脑力竞技** —— 两名玩家速答题目互相打伤害,每段抽 3 选 1 强化构筑流派,
强化越叠越离谱,最后进入狂暴期决出胜负。

> 不是"学习版 MOBA",而是"答题驱动的实时构筑对战"。学习是载体,**爽是目的**。

### 设计支柱(三条,排序即优先级)
1. **打击感** —— 答对的瞬间反馈必须脆、必须爽。这是留存的命根子。
2. **构筑深度** —— "复杂的是 Build,不是操作"。每局不同,源于强化联动。
3. **公平博弈** —— 1v1 对称,无付费数值,胜负取决于答题 + 构筑决策。

---

## 二、核心玩法循环

玩家在局内**只做两件事**:
1. **答题** → 产出 伤害 / 能量 / 连击
2. **抽强化** → 决定本局流派,强化自动联动,无需操作

```
答对 ──► 打伤害 + 攒能量 + 连击+1
  │         │            │
  │         ▼            ▼
  │     能量满→放技能   连击档位→伤害倍率(3连×1.4 / 5连×2.0)
  ▼
段落结束 ──► 3选1 强化 ──► Build 逐渐成型 ──► 狂暴期全部翻倍 ──► 结算
```

### 答题规则(本版:数学速算)
- 题目为四选一速算(乘法 / 加减 / 两步混合),难度随对局推进上升。
- **答对**:连击 +1,能量 +15,按公式对对手造成伤害。
- **答错**:连击清零(除非有容错强化),自身受小额惩罚(-40 HP)。
- **越快越狠**:答题耗时影响"极速演算"类强化的加成。

### 胜负判定
- 任一方 HP 归零 → 立即 KO 胜。
- 计时结束双方都存活 → **剩余 HP 高者胜**。
- 设计意图:既保留"一波带走"的爆发快感,也保证每局必定 3 分钟内结束。

## 三、对局节奏(本版:同步对战,每题一回合)

为聚焦"交互 + 构筑",本版采用**同步对战**:**每道题就是一个回合,双方同时作答**——
你对了打它、它对了打你。这比"你打完一轮再轮到 AI"更有对抗感(参考头脑王者的同步答题)。

- **原子单位 = 一题**。双方面对同一道题,你答完立刻结算双方结果。
- **连击跨题累积**(不再按回合清零),让连击 / 构筑流真正滚雪球。
- **4 小局,每局 5 题(共 20 题)。开打前抽,3 次抽取**;最后一小局狂暴(全部 ×2)。
- 胜负:任一方血量归零即 KO;或答满判血量高者胜。

| 节点 | 抽取 | 档位 | 构筑状态 | 情绪曲线 |
|---|---|---|---|---|
| 抽取 1 → 第 1 小局 | ✅ | 🩶 银 · **催化剂** | 选定方向 | 期待 |
| 抽取 2 → 第 2 小局 | ✅ | 🟡 金 · **收益** | 联动初现 | 投入 |
| 抽取 3 → 第 3 小局 | ✅ | 💠 棱彩 · **核心** | 引擎成型 | 上头 |
| 🔥 第 4 小局(狂暴) | — | 全部强化 ×2 | 失控滚雪球 | 高潮 |
| 结算 | — | — | 高光 + Build 总结 | 释放 |

> 节奏铁律(同步版):**每题都是一次正面对抗,3 次抽取各对应构筑的一个阶段**
> (催化剂→收益→核心)。短反馈(每题胜负)嵌在长决策(凑 Build)里。
> 每次抽取还可 **重抽 2 次**——主动去"找"你的流派(掌控感,见第四章)。
>
> 关键机制(成败点):**强化要越来越离谱**。第 1 张是 +60%/护盾这类温和加成,
> 到狂暴回合所有效果翻倍、触发链全屏炸开,让玩家进入"我要成神了"的状态。
>
> **构筑可视化**(本版核心体验):答对题时,屏幕实时弹出触发的强化名与放大倍率
> (`超频核心 ×1.6`、`暴击!`、`真理穿透 ×2`),底部 Build 栏对应强化高亮闪动。
> 目的:让"Build 成型"这件事**肉眼可见、即时可感**,这是回合制版要验证的第一爽点。
>
> **对手透明化**(策略博弈基础):双方信息对称——(1) 抽强化时显示对手当前 Build 与准确率,
> 让玩家**针对性 counter-draft**;(2) 底栏并排显示"你 vs 对手"两套 Build;(3) AI 回合**逐题播报**
> (答对/答错、连击、暴击、护盾挡了多少、对你造成多少)。玩家能据此随时调整战略:
> 对面暴击流→抢护盾/反射;对面老答错→加大污染/续航。没有信息差,胜负才落在构筑决策上。
>
> (实时版节奏作为备选保留:`40s 紧张 → 决策 → 狂暴爆发`。手感验证通过后再考虑是否回归实时。)

## 四、海克斯强化系统(核心灵魂)

### 4.0 上瘾机制调研(2026-06-27,二轮:Balatro / 杀戮尖塔 / 吸血鬼幸存者)
首轮只看了斗魂竞技场,联动还是"加法",**不够上头**。二轮研究 Balatro 等构筑成瘾标杆,找到真正的钥匙:

1. **底数 × 倍率,且关键在"×倍率"而非"+倍率"**(Balatro 核心):
   - 两张 +30 = +60(加法,平);一张 +30 配 **×2** = 120,再配 **×3** = 360;
   - **连乘三个 ×倍率,三手内从几千飙到几百万。** 指数爆炸才是上瘾源。
2. **期待 > 奖励**:多巴胺在结算那一刻爆发——需要"跑分"动画把算术变成放烟花。
3. **逐项结算、因果可见**:每张牌依次跳动、跑分实时更新,玩家直觉理解谁和谁联动。
4. **永远有墙逼你**(Balatro 的盲注每关 ×1.5–2 涨):本作靠"对手逐局变强"实现,你不能躺。
5. **重抽 + 随机性**:主动找 Build 的掌控感 + 老虎机心理 + 每局不同。

> **诊断**:此前所有版本都是"+x%"加法堆叠 → 曲线要么平要么乱,这同时造成了"不上头"和"数值不对"。
> v0.6 改用底数×倍率,从根上解决。

### 4.1 计分模型:伤害 = 底数 × 倍率
每次答对的伤害按 Balatro 模型结算,**屏幕显示跑分**:`底数 32 × 倍率 6.4 = 205 💥`。

- **底数(chips)** = 基础 + 连击×系数 + 加底数强化(护盾转底数等)。这是"加起来"的部分。
- **倍率(mult)** = `(1 + Σ+倍率) × Π×倍率`。**×倍率连乘**是指数爆炸与上瘾的来源。
- 狂暴小局:倍率再 ×2。

强化按对计分的作用分三类(卡面标注):
| 类别 | 作用 | 例 |
|---|---|---|
| 🔵 **+底数** | 加底数,线性、稳 | 锐刃、节奏、护盾转化 |
| 🟡 **+倍率** | 加进 (1+Σ),中等 | 积怨、势如破竹 |
| 🟣 **×倍率** | **连乘**,指数、上瘾 | 暴击增幅、超频核心、真理穿透、熔毁、无限超载 |

### 4.2 牌的三种角色:催化剂 → 收益 → 核心
**3 次抽取对应一条 Build 链**(对齐银/金/棱彩三档):

| 抽取 | 档位 | 角色 | 作用 |
|---|---|---|---|
| 第 1 抽 | 🩶 银 | **催化剂** | 启动方向(叠过载 / 起暴击 / 起连击 / 起护盾 / 起能量) |
| 第 2 抽 | 🟡 金 | **收益** | 放大它(+倍率 / 第一个 ×倍率 / 暴击喂连击…) |
| 第 3 抽 | 💠 棱彩 | **核心** | ×倍率引擎,打破上限自我滚雪球 |

**唯一性 + 重抽**:抽过的不再出现;每局 **2 次重抽**主动找牌。
**联动偏好**:发牌按已有流派标签**加权**(像斗魂竞技场"倾向你当前 Build"),让 3 张也能凑成链。

### 4.3 主要流派(催化剂→收益→核心,跨系可嫁接)
- **过载流** = 过载印记(叠层)→ 积怨(+倍率/层)/引爆(暴击炸光)→【熔毁】(无上限,每 10 层 ×1.6 连乘)
- **暴击引擎** = 暴击芯片 → 暴击增幅(暴击 ×1.6)/暴击回响(暴击+连击)→【无限超载】(暴击后必暴 + 额外 ×1.5):必暴链滚乘
- **连击雪球** = 节奏(连击+底数)→ 势如破竹(+倍率/连击)/超频核心(连击≥5 ×1.8)→【奇点】(连击不灭 + 连击倍率系数翻倍)
- **护盾转化** = 知识护盾 → 护盾转化(护盾→底数)→【永动核心】:越苟底数越高
- **能量爆发** = 充能 → 能量虹吸 →【超频奇点】:技能按当前倍率计分,大招也吃 ×倍率
- **污染瘫痪** = 认知污染 → 虹吸 →【全面污染】:对面答不出题,失误奶你+叠过载

> **跨系嫁接**:引爆把"过载"接到"暴击";真理穿透让"难题"成为 ×倍率开关;
> 暴击增幅 + 无限超载 + 超频核心 三个 ×倍率连乘 = 单题伤害指数起飞。

> 关键张力:**积怨(持续 +倍率) vs 引爆(一次性炸光)**——同样叠过载,两种打法。

> 平衡红线:**不能出现"必选核心"**。每条流派要有 counter。需大量对局验证。

### 4.4 数值平衡 + 调参面板(v0.6)
**根因修复**:此前用加法堆叠 → 曲线平/乱,这同时是"不上头"和"数值不对"的原因。
v0.6 改 **底数×倍率**:底数线性兜底,×倍率连乘提供爆发。曲线天然"前期稳、后期炸",更好调也更上头。

**平衡目标**:对局通常打到**第 3–4 小局**,**狂暴小局决胜**;到终局双方血量接近见底。
专精 ×倍率 引擎应在狂暴局提前 KO(体现构筑),散构筑接近五五开。

**调参面板(原型内置 ⚙)**:数值无法靠静态推演定准,所以做成**边玩边拨**的滑块,改完即生效。基线值:

| 旋钮(面板可调) | 基线 | 说明 |
|---|---|---|
| 双方血量 `hp` | 2200 | ×倍率 爆发后伤害跨度大,血量需更高 |
| 玩家基础底数 `baseChips` | 12 | 每题底数起点 |
| 连击底数 `comboChips` | 3 | 连击每层 +底数 |
| 大招基础 `ultBase` | 200 | ×狂暴(×2);超频奇点 额外吃倍率 |
| 对手基础伤害 `foeBase` | 60 | — |
| 对手逐局成长 `foeRamp` | 0.45 | 每小局 ×(1+0.45);对手没引擎,靠爬升保持威胁 |
| 对手准确率 `foeAcc` | 0.72 | 被污染下降 |

> 注:**静态只能定起点,真正平衡看对局数据**——跑 20+ 局记录各流派胜率/平均结束小局,据此拨面板。
> 这一版把"调"的权力直接交给玩家/设计者,迭代不再依赖改代码。

> 注:**静态测算只能定起点,真正平衡要看对局数据**。下一步应跑 20+ 局记录各流派胜率与平均结束小局,
> 据此微调上表旋钮,消除必选核心 / 过弱流派。

## 五、打击感清单(支柱一,必须做到)

- **数值飘字**:答对弹金色数字,暴击放大 + 变色。
- **连击横幅**:5/10/15 连击全屏播报。
- **暴击震屏 + 音效**:暴击/技能/绝杀有独立音效与屏幕抖动。
- **能量爆发**:能量满自动放技能 = 全屏爆发 + 重音。
- **狂暴期氛围**:背景变红、内发光、出题加速,听觉/视觉同步升级。
- **结算高光**:自动识别"最高连击 / 单次最高伤害 / 技能次数 / 清空血条"生成一句高光文案。

> 原型已实现以上全部。本版迭代重点是**手感打磨**,不是加功能。

## 六、本版边界(明确不做)

| 不做 | 原因 |
|---|---|
| 3v3 / 联机 | 工程地狱,且不产生乐趣。先用 1v1 验证核心。 |
| 真实题库 / 多学科 | 数学速算足够验证节奏。题库是资产问题,不是设计问题。 |
| 账号 / 段位 / 商业化 | 留存与变现是"爽了之后"的事。 |
| 固定职业 | 已被 5.5 决策否决,改为局内强化决定流派。 |

## 七、验证目标(回合制版要回答的三个问题)

> 本版的第一验证目标是**构筑**,不是手感。

1. **抽强化时,玩家有没有"想凑某流派"的冲动?**(支柱二 / 本版核心)
2. **Build 成型那一刻("触发链全屏炸开")爽不爽?**(构筑可视化是否成立)
3. **狂暴回合是不是真有高潮感?**(节奏曲线是否成立)

三个都"是" → PROCEED:扩强化池 + 调平衡,再考虑加回实时 / 3v3 / 题库。
有"否" → 先在回合制 1v1 这层把构筑调到爽,再往上加。
