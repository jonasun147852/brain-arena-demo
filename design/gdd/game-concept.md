# 游戏概念 · 1v1 核心版

> 状态:Draft v0.2 · 2026-06-27(回合制改版)
> 范围:1v1 单机/对战核心循环。3v3、题库、联机、商业化均**不在本版**。
> 本版方向:**回合制 + 聚焦构筑**。先验证"凑 Build"够不够爽,手速/实时延后。
> 这份文档配套可玩原型:`prototypes/core-loop/index.html`,设计与代码同步演进。

---

## 一、一句话概念

**实时 Roguelike 脑力竞技** —— 两名玩家速答题目互相打伤害,每段抽 3 选 1 强化构筑流派,
强化越叠越离谱,最后进入狂暴期决出胜负。

> 不是"学习版 MOBA",而是"答题驱动的实时构筑对战"。学习是载体,**爽是目的**。

### 设计支柱(三条,排序即优先级)
1. **打击感** —— 答对的瞬间反馈必须脆、必须爽。这是留存的命根子。
2. **构筑深度** —— "复杂的是 Build,不是操作"。每局不同,源于强化联动。
3. **公平博弈** —— 1v1 对称,无付费数值,胜负取决于答题 + 构筑决策。

---

## 二、核心玩法循环

玩家在局内**只做两件事**:
1. **答题** → 产出 伤害 / 能量 / 连击
2. **抽强化** → 决定本局流派,强化自动联动,无需操作

```
答对 ──► 打伤害 + 攒能量 + 连击+1
  │         │            │
  │         ▼            ▼
  │     能量满→放技能   连击档位→伤害倍率(3连×1.4 / 5连×2.0)
  ▼
段落结束 ──► 3选1 强化 ──► Build 逐渐成型 ──► 狂暴期全部翻倍 ──► 结算
```

### 答题规则(本版:数学速算)
- 题目为四选一速算(乘法 / 加减 / 两步混合),难度随对局推进上升。
- **答对**:连击 +1,能量 +15,按公式对对手造成伤害。
- **答错**:连击清零(除非有容错强化),自身受小额惩罚(-40 HP)。
- **越快越狠**:答题耗时影响"极速演算"类强化的加成。

### 胜负判定
- 任一方 HP 归零 → 立即 KO 胜。
- 计时结束双方都存活 → **剩余 HP 高者胜**。
- 设计意图:既保留"一波带走"的爆发快感,也保证每局必定 3 分钟内结束。

## 三、对局节奏(本版:回合制,5 回合)

为聚焦"构筑"而非手速,本版改为**回合制**:玩家有时间观察自己的 Build 成型、思考下一张抽什么。

一回合 = **你的回合(答 5 题)→ 结算伤害 → AI 回合(答 5 题)→ 抽 1 张强化**。

| 回合 | 内容 | 构筑状态 | 情绪曲线 |
|---|---|---|---|
| 开局 | 3 选 1 抽初始强化(定流派起点) | 种子 | 期待 |
| 第 1 回合 | 双方各答 5 题,只有 1 张强化 | 雏形 | 升温 |
| 第 2 回合 | 第 2 张到手,联动初现 | 显形 | 投入 |
| 第 3 回合 | 第 3 张,流派定型(暴击/护盾/污染) | 成型 | 上头 |
| 第 4 回合 | Build 滚雪球,触发链变长 | 爆发 | 高压 |
| 🔥 狂暴回合(第 5) | **所有强化效果 ×2**,背景变红 | 失控 | 高潮 |
| 结算 | 胜负 + 高光时刻 + MVP 数据 | — | 释放 |

> 节奏铁律(回合制版):**每回合一张强化 = 一次明确的构筑决策**。玩家每回合都能
> 清楚看到"我这张选下去,Build 会怎么变强"——决策的重量取代了手速的紧张。
>
> 关键机制(成败点):**强化要越来越离谱**。第 1 张是 +60%/护盾这类温和加成,
> 到狂暴回合所有效果翻倍、触发链全屏炸开,让玩家进入"我要成神了"的状态。
>
> **构筑可视化**(本版核心体验):答对题时,屏幕实时弹出触发的强化名与放大倍率
> (`超频核心 ×1.6`、`暴击!`、`真理穿透 ×2`),底部 Build 栏对应强化高亮闪动。
> 目的:让"Build 成型"这件事**肉眼可见、即时可感**,这是回合制版要验证的第一爽点。
>
> (实时版节奏作为备选保留:`40s 紧张 → 决策 → 狂暴爆发`。手感验证通过后再考虑是否回归实时。)

## 四、技能 / 强化系统(核心灵魂)

每局共 **4 次** 3 选 1 抽取(开局 + 3 段)。强化分三系,可叠加,自动联动成 Build。

### 输出系(Damage)—— 爽 / 暴击 / 连击
| 强化 | 效果 |
|---|---|
| 超频核心 | 连击≥3 时,每次答对额外 +60% 伤害(可叠) |
| 暴击芯片 | 暴击率 +25%,暴击伤害 +60% |
| 连锁爆破 | 每次暴击额外造成一次 40% 溅射 |
| 极速演算 | 答得越快伤害越高(3 秒内最高 +80%) |
| 真理穿透 | 难题(困难标记)伤害翻倍 |

### 防御系(Support)—— 稳定 / 续航 / 翻盘
| 强化 | 效果 |
|---|---|
| 知识护盾 | 每连对 5 题获 120 护盾,优先抵伤 |
| 容错矩阵 | 答错不清空连击(每段最多 2 次) |
| 稳态回路 | 答对额外 +8 能量;能量满自动放技能(全屏爆发) |
| 知识汲取 | 造成伤害的 15% 转为自身回血 |

### 干扰系(Control)—— 博弈 / 恶心 / 翻盘
| 强化 | 效果 |
|---|---|
| 认知污染 | 敌方答对率 -12% |
| 逻辑迟滞 | 敌方答题速度 -18% |
| 精神过载 | 敌方连击越高越易被打断(清空其连击) |

### 强化联动 = Build(真正上瘾点)
真正上瘾的不是单个强化,而是**强化之间凑成流派**:

- **无限暴击流** = 暴击芯片 + 连锁爆破 + 极速演算 → 越打越快、越快越暴、越暴越多溅射
- **永动护盾流** = 知识护盾 + 稳态回路 + 知识汲取 → 苟住续航,狂暴期反杀
- **精神污染流** = 认知污染 + 逻辑迟滞 + 精神过载 → 对面根本答不出题

> 平衡红线(1v1 对称竞技最大风险):**不能出现"必选强化"**。一旦某张无脑最优,
> 流派多样性立刻崩塌。每系都要有能 counter 另一系的路径(盾 克 爆发,干扰 克 连击,爆发 克 苟)。
> 这是后续数值调参的核心目标,需要大量对局验证。

## 五、打击感清单(支柱一,必须做到)

- **数值飘字**:答对弹金色数字,暴击放大 + 变色。
- **连击横幅**:5/10/15 连击全屏播报。
- **暴击震屏 + 音效**:暴击/技能/绝杀有独立音效与屏幕抖动。
- **能量爆发**:能量满自动放技能 = 全屏爆发 + 重音。
- **狂暴期氛围**:背景变红、内发光、出题加速,听觉/视觉同步升级。
- **结算高光**:自动识别"最高连击 / 单次最高伤害 / 技能次数 / 清空血条"生成一句高光文案。

> 原型已实现以上全部。本版迭代重点是**手感打磨**,不是加功能。

## 六、本版边界(明确不做)

| 不做 | 原因 |
|---|---|
| 3v3 / 联机 | 工程地狱,且不产生乐趣。先用 1v1 验证核心。 |
| 真实题库 / 多学科 | 数学速算足够验证节奏。题库是资产问题,不是设计问题。 |
| 账号 / 段位 / 商业化 | 留存与变现是"爽了之后"的事。 |
| 固定职业 | 已被 5.5 决策否决,改为局内强化决定流派。 |

---

## 七、验证目标(回合制版要回答的三个问题)

> 本版的第一验证目标是**构筑**,不是手感。

1. **抽强化时,玩家有没有"想凑某流派"的冲动?**(支柱二 / 本版核心)
2. **Build 成型那一刻("触发链全屏炸开")爽不爽?**(构筑可视化是否成立)
3. **狂暴回合是不是真有高潮感?**(节奏曲线是否成立)

三个都"是" → PROCEED:扩强化池 + 调平衡,再考虑加回实时 / 3v3 / 题库。
有"否" → 先在回合制 1v1 这层把构筑调到爽,再往上加。
