## A. 机制选型(唯一方案)

**「鸿运局」— 运行级(run-level)发牌修饰,数值骨架零改动。**

- **触发**:上一局 finishRun(胜负皆算,弃局/刷新不算)时为下一局独立 roll,`p = 0.06`。**无 pity、无计数器、无袋。** 两条唯一干预:①刚打完鸿运局,下一局锁 0%(防连爆通胀参照点);②RUNS≥2 才参与(首局教学窗干净 + 彩池 7 张齐装后才配开神局)。
- **一次性种子**:第 2 局打完时若从未见过鸿运,强制下一局(第 3 局)为鸿运,`ba_luckSeeded` 记一次永不复发。这不是保底,是单机版的"邻座中奖"——让玩家知道彩票真实存在的唯一可靠办法。
- **机制内容**(全部在发牌层):
  1. 四次强化抽换用**鸿运轮色表**(全程无银、彩收尾):30s `金65/彩35` · 75s `金30/彩70` · 2min `金10/彩90` · 3min `彩100`。
  2. **重抽 1 → 2 次**(纯 agency,零数值)。
  3. 池干涸兜底在鸿运局内**先彩后金**降级(不许随机乱补,保脸面)。
  4. 其余一切照旧:开局立派、觉醒独立 roll 与 pity、天命翻牌、赌卡、敌方血池 [180,400,680]、能量公式,一个数不动。
- **宣告演出**(只在中奖时播,揭晓点=首个强化抽开屏,战斗内或过关补给屏皆可触发):
  bDraft/secDraft 加 `.lucky` 类(鎏金边+金尘粒子)→ 横幅「**🌠 鸿运当头!这把,张张是好牌**」→ 专属上行琶音 SFX.lucky(禁复用 tierUp)→ HUD 剩余发牌刻度全部镀金(期待的物理载体)→ draftRoundTag 全程追加「· 🌠鸿运」→ 战报金框徽记。
- **彩闪(基线局的小奖梯)**:普通局 `p = 0.12` 至多一次,随机命中 R2 或 R3 那一手,该手轮色表下移一行(+1 档);开屏 toast「**✨ 彩闪!这一手,亮一档**」,金光闪一下(不烧全程)。鸿运局内不 roll。它是独立小奖,**永不**被 framing 成"差一点鸿运"。

## B. 频率与感知

- **平均频率**:自然 1/17 局(约每 1-2 个晚上一次);种子保证第 3 局必见一次;彩闪约 1/8 局给局间方差铺纹理。
- **没中的把怎么知道神局存在**(四条诚实通道,零比惨):
  1. 第 3 局种子 = 亲身体验过一次;
  2. 标题屏 bestLine 下加一行「🌠 鸿运 ×N」(累计奖杯,**永不**显示"多少局没中");
  3. 标题屏副文案一句松散公示:「听说每十七把,就有一把鸿运当头」;
  4. 彩闪与鸿运同语系降半格——视觉语言被预习过,真撞上时零延迟读懂。
- **近失的度:零。** 普通局不播"差一点"、不展示"你本来会拿到什么"、无概率计、无连没计数(Universal 1988 红线 + 参照点管理)。允许存在的唯一"近失"是真实的:彩闪是它自己的胜利,不是失败的鸿运。

## D. 功率预算

- **目标增量 +25~35%**(研4 蒙特卡洛:M1.3→SKILLED 97.1/AVERAGE 36.1;M1.5→97.4/37.6;禁区 M≥1.75)。鸿运表期望彩手 ≈2.95(基线 1.1),落 M≈1.35-1.5 带内:SKILLED 败率减半但仍死于连错(断连/挨打/过载崩塌全不随神局缩放),AVERAGE 通关率约翻倍但打烂照输——**神局≠必胜**。
- **封顶**:不加新帽,现有四道全部保留且不豁免——溢出结转 ≤下阶段预算 50%(L2039-2045)、攻击加法 16 帽、溅射 0.9 帽、OVERLOAD_MAX 5。鸿运局单发理论峰值 ~800-900(670 双窗 × 卡质增量),硬验收 ≤900。
- **耦合处理**:①觉醒/pity 完全不碰(觉醒独立于轮色,L1339-1348;"养出来的赌局"与"彩票"两套逻辑绝不共用对象);pity 保底局偶遇鸿运(≈0.4%)是故事不是 bug,放行。②快玩家兜底零改动:DRAFT_FLOOR 欠牌走同一 rollOffer,鸿运轮色自动带上;终局追赶把末手压进收尾 12s = 天然自限负反馈,保留。③敌方难度、rubber-band:一个字不动(神局内暗调难度 = 归因滑向"游戏坑我",流失)。

## F. 验收线(§13 遥测 + 研4 模拟脚本复跑,scratchpad/godrun_sim.js)

1. **出现率**:2000 局模拟(RUNS≥3、剔除种子),鸿运局占比 ∈ [4.5%, 7.5%];连续两局鸿运出现次数 = 0。
2. **神局≠必胜**:鸿运局通关率 SKILLED ∈ [90%, 98%](断言 <100%,500 局内至少 1 败);AVERAGE ≤ 60%。
3. **封顶**:鸿运局单发伤害 max ≤ 900,P(>900) = 0;四道帽触发路径与基线一致。
4. **基线不回退**:`R.lucky=false && flashAt=null` 路径下轮色分布与 v5.1.1 逐位一致(diff 断言);含彩闪的普通局全局输出增量 ≤ +8%,通关率漂移 ≤ ±2pp。
5. **池不漏气**:鸿运局四手中触发"任意色兜底"的手占比 ≤ 2%(先彩后金降级生效)。
6. **两套彩票隔离**:觉醒出现率与 pity 分布在鸿运/普通局间无统计差异(KS 检验 p>0.05 量级即可)。
7. **反刷取**:弃局/刷新 10000 次模拟,ba_luck 置位次数 = 0;鸿运局内刷新后 R.lucky 不复活。

**不做清单**(与实现同权重):无保底/pity/袋;无概率计与"没中"播报;无假近失与"差一点"动画;鸿运视觉语言不外借;鸿运输局无安慰奖;鸿运内不动难度。
