## 0. 一句话定位与设计支柱

**实时 Roguelike 脑力竞技**:答题驱动的实时构筑对战。学习是载体,**爽是目的,公平是底线**。

支柱(排序即优先级,冲突时按此裁决):
1. **构筑成瘾** —— 凑 ×倍率 引擎、滚雪球,让人"再来一局"。
2. **打击感** —— 答对=出招打中对手,反馈脆、爆、即时。
3. **公平博弈** —— 对称竞技、无付费数值、服务器权威、反作弊。
4. **被需要(团队层)** —— 3v3 里每个角色都被队友需要(原 PDR 的核心情感)。

> 任何新增内容,先问:它服务哪条支柱?不服务的砍掉。

## 1. 核心循环(moment-to-moment)

```
看题 → 答对 → [底数 × 倍率] 结算伤害 → 出招命中对手 → 反馈爆炸
  └ 答对还产出:连击+1 / 能量 / 过载层(取决于你的 Build)
每 5 题(一小局)结束 → 3选1 抽海克斯强化(可重抽)→ Build 进化
4 小局后 / 一方血量归零 → 结算(MVP·高光·Build 回顾)
```

**两件事原则**(降低操作门槛,深度在 Build 不在操作):玩家全程只做 **① 答题 ② 抽强化**。强化效果自动联动。

## 2. 题目系统(作为玩法机制,不只是内容)

题目不是"考试",是**战斗节奏的脉搏 + 构筑的触发器**。

### 2.1 难度分层 = 风险/收益
- **普通题**:基础底数,稳。
- **难题(✦)**:更难,但触发"难题系"×倍率(如真理穿透 ×2.2)。**难题是高风险高回报的 ×倍率 开关**,不是单纯加分。
- 难题出现率随小局推进上升(后期更刺激)。

### 2.2 自适应难度(Adaptive)
- 按玩家近 N 题正确率动态调档,保证"踮脚够得着":太顺→加难,连错→缓一下。
- 竞技公平:**1v1/3v3 双方同题同档**(对称),自适应只用于排位匹配前的"学力评估"和 PVE/训练模式。

### 2.3 动态参数化题库(反作弊核心资产)
- 题目以**模板 + 参数**存储(如 `a × b + c`,运行时随机 a/b/c),**不是静态题面**。
- 作用:① 同一模板生成无限变体,**搜题/背答案失效**;② 难度可按参数精确控制;③ 数据干净(知道考点)。
- 这是平台竞技公平的**地基**,从第一天就要这么建库(原 PDR 已点明,正确)。

### 2.4 学科无关化
- 计分/构筑系统**完全不依赖学科**:答对=产出,管你是速算、单词、还是语法。
- 好处:换题库即换学科,玩法零改动;先用速算验证,后扩科。

## 3. 海克斯构筑系统(系统核心)

### 3.1 计分:底数 × 倍率(Balatro 模型)
`伤害 = 底数(chips) × 倍率(mult)`,屏幕**跑分可见**。
- **底数** = 基础 + 连击×系数 + 加底数强化。线性兜底。
- **倍率** = `(1 + Σ+倍率) × Π×倍率`。**×倍率连乘 = 指数爆炸 = 上瘾源。**

强化三类(卡面标注):
| 类别 | 数学作用 | 心理作用 |
|---|---|---|
| 🔵 +底数 | 加常数 | 稳、可靠,新手友好 |
| 🟡 +倍率 | 加进 (1+Σ) | 中等成长,看得见涨 |
| 🟣 **×倍率** | **连乘** | 指数、上头、"我要成神" |

### 3.2 三档品质 = 三段构筑(银→金→棱彩)
| 抽取 | 档 | 角色 | 定位 |
|---|---|---|---|
| 第 1 | 银 | **催化剂** | 选定方向(起暴击/连击/过载/护盾/能量/干扰) |
| 第 2 | 金 | **收益** | 放大它(+倍率 / 第一个 ×倍率 / 喂资源) |
| 第 3 | 棱彩 | **核心** | ×倍率 引擎,打破上限自我滚雪球 |

**重抽**:每局 2 次,主动找 Build(掌控感 + 老虎机)。**唯一性**:抽过不再现,逼取舍。
**联动偏好发牌**:按已有标签加权,3 张也能凑成链(学斗魂竞技场"倾向当前 Build")。

### 3.3 流派矩阵(每条 = 催化剂→收益→核心)
| 流派 | 催化剂(银) | 收益(金) | 核心(棱彩) | 爽点 |
|---|---|---|---|---|
| 暴击引擎 | 暴击芯片 | 暴击增幅(×1.6)/暴击回响 | 无限超载(必暴+×1.5) | 必暴链连乘 |
| 连击雪球 | 节奏 | 势如破竹/超频核心(×1.8) | 奇点(连击不灭·系数翻倍) | 连击无限滚 |
| 过载流 | 过载印记 | 积怨(+倍率/层)/引爆 | 熔毁(无上限·每10层×1.6) | 层数滚雪球 |
| 攻防一体 | 知识护盾 | 护盾转化(护盾→底数) | 永动核心 | 越苟越猛 |
| 能量爆发 | 充能 | 能量虹吸 | 超频奇点(技能吃倍率) | 大招连发 |
| 污染瘫痪 | 认知污染 | 虹吸/乘虚而入 | 全面污染 | 对面答不出题 |

**跨系嫁接是乐趣来源**(关键):引爆把"过载"接到"暴击";真理穿透让"难题"成 ×倍率 开关;
护盾转化把"防御"接成"输出"。三系不是孤岛——目标是**几十种可凑引擎**。

### 3.4 构筑深度的扩展方向(PROCEED 后)
- **每系催化剂/收益扩到各 3-4 张**(现在偏少),让 3 选 1 真有取舍。
- **配方/发现**(背包乱斗启发):某些强化组合解锁隐藏效果(如 暴击+过载 → "暴击叠双倍过载"),奖励试验。
- **负面/诅咒强化**(高风险高回报):接受一个 debuff 换强力 ×倍率,制造决策张力。

## 4. 平衡框架(竞技平台最难、最关键)

> 1v1/3v3 对称竞技,**最大死法 = 出现"必选核心"或"必胜流派"**,流派多样性一崩,游戏就死。

### 4.1 结构性约束(从设计上防失衡)
1. **限制构筑件数**(超级自走宠的核心智慧):只 3 件强化 → 组合可控、可穷举测、好平衡。**不要为了"内容多"而加抽取次数。**
2. **counter 三角**:每条流派都要被另一条克——
   - 护盾/续航 **克** 暴发(苟过爆发期)
   - 干扰/污染 **克** 连击(打断连击链)
   - 暴发/斩杀 **克** 苟(在对方滚起来前打死)
   - → 形成"石头剪刀布",没有无脑最优。
3. **×倍率 软上限**:同类 ×倍率 收益递减或设触发条件(如超频需连击≥5),防止纯堆乘炸穿。

### 4.2 数据驱动调参(运营期)
- 每个赛季跟踪:**各流派胜率、首选率、平均结束小局、棱彩选取分布**。
- 目标带:任何流派胜率落在 **45%–55%**;无流派首选率 > 35%。
- 偏离 → 拨数值旋钮(原型已内置调参面板,迭代不靠改代码)。

### 4.3 赛季刷新(TFT 智慧)
- 每赛季**轮换部分强化池 + 加新机制**,既保鲜又自然再平衡(把强势流派换下去)。
- 题库主题也可季节化(节日题、学科活动)。

## 5. 单局结构与节奏

| 节点 | 抽取 | 倍率环境 | 情绪 |
|---|---|---|---|
| 抽取1 → 第1小局 | 银·催化剂 | 基线 | 选定方向 |
| 抽取2 → 第2小局 | 金·收益 | 联动初现 | 投入 |
| 抽取3 → 第3小局 | 棱彩·核心 | 引擎成型 | 上头 |
| 🔥第4小局 狂暴 | — | **全部 ×2** | 高潮 |
| 结算 | — | — | 释放 |

- **每题一回合,双方同步答同题**(对抗感)。连击跨题累积(雪球)。
- 胜负:血量归零即 KO;答满判血量高者胜。
- **狂暴局**是决胜设计:专精引擎应在此提前 KO,体现构筑;散构筑则五五开拼操作。
- 目标单局时长:**3–5 分钟**(适合碎片/课间)。

## 6. 1v1 → 3v3 团队对战(核心扩展)

> 团队层不是"三个 1v1 并排",而是**把"催化剂→收益"从局内扩展到玩家之间**(Arena 双人智慧:一人 setup,一人收割)。

### 6.1 跨玩家协同(团队 build 的灵魂)
- 部分强化的收益**作用于队友**:
  - 【过载印记】队友答对给敌方叠层 → 你的【积怨/引爆】吃**全队叠的层数** → 一人当"上分员"叠层,一人当"爆发位"收割。
  - 【思维同步】队友答对,你也得能量 → 喂养爆发位的大招。
  - 【知识共鸣/团队护盾】辅助位答对给全队上盾 → 让爆发位浪。
- → **角色从 Build 涌现,而非固定职业**:有人天然走"叠层/辅助",有人走"爆发/收割"。

### 6.2 团队资源与情感钩子(被需要)
- **团队能量槽 / 合体技**:三人短时间内连续答对触发"三位一体"大招(原 PDR 的同步共振)。
- **救赎机制**:队友答错时,你做对"补位题"抵消其惩罚 → 系统播报"XXX 救助了你" → **情感粘性 > 单纯加分**。
- 这条直接实现原 PDR 最强留存点:**"被队友需要"**。

### 6.3 社交护城河
- **组队开黑奖励**(Arena Fame 智慧):与好友固定队额外加成 → 熟人社交 = 最强防流失(原 PDR 已点明)。
- **双段位**:个人段位(学力)+ 团队段位(默契),分别排名,鼓励拉好友。

### 6.4 团队平衡注意
- 跨玩家 ×倍率 联动必须**有上限/递减**,否则三人叠同一引擎=指数中的指数,直接炸穿。
- 团队 counter 三角同样成立:全爆发队怕全干扰队,等等。

## 7. 元进度与赛季(平台层)

> 仅作方向,细节待商业化文档。**铁律:皮肤/通行证只卖外观,绝不卖数值**(竞技公平 + 防沉迷/双减合规)。

- **段位天梯**:个人学力段位 + 团队默契段位(青铜→王者),赛季重置。
- **海克斯图鉴/解锁**:通过对局逐步解锁更多强化进池(给新手平滑曲线 + 长期目标)。
- **赛季通行证**:任务驱动(本周赢 X 局、达成某流派),奖励皮肤/外观/称号——**非数值**。
- **皮肤系统**:角色皮、技能特效皮、"知识外观"。是变现主力,也是 Cocos/Spine 的舒适区。
- **高光时刻自动剪辑**:绝杀/连对/翻盘 一键分享(原 PDR,社媒裂变引擎)。
- **家长端**:庄重专业的"竞技成长报告"(四维:逻辑/速度/抗压/协作),消除游戏偏见。

## 8. 防作弊与公平(竞技平台地基)

| 威胁 | 对策 |
|---|---|
| 搜题 / 背答案 | **动态参数化题库**(§2.3),题面无限变体 |
| 改客户端伤害/计分 | **服务器权威**:答对判定、计时、计分、伤害全在服务器算,客户端只显示 |
| 脚本自动答题 | 行为检测(答题间隔分布异常)、人机验证、参数题需真算 |
| 段位代打/匹配操纵 | MMR 异常检测、设备指纹 |

> 这些是"正经对战平台"和"小游戏"的分水岭。**计分逻辑从第一天就要设计成可搬到服务器**(原型里 `computeDamage` 是纯函数,正是为此)。

## 9. 内容路线图(玩法视角)

| 阶段 | 玩法范围 | 目标 |
|---|---|---|
| **0 验证**(现在) | 单机 1v1 vs AI,速算,3 流派引擎 | 证明构筑上瘾 + 打击感(再来一局率) |
| **1 在线 1v1** | 真人 1v1,服务器权威计分,动态题库 | 验证留存(次留/7留)、获客成本 |
| **2 排位** | MMR 匹配、个人段位、赛季 | 验证竞技拉力 + 长期留存 |
| **3 团队 3v3** | 跨玩家协同、救赎、合体技、双段位、组队 | 上社交护城河(熟人留存) |
| **4 内容化** | 赛季刷新、强化扩池、皮肤、通行证、多学科 | 保鲜 + 变现 |

> **纪律**:每阶段数据达标才进下一阶段。别在验证好玩前砸平台。

## 10. 开放设计问题(需决策 / 待playtest)

1. **单局时长 3 还是 5 分钟?** 课间碎片(3)vs 构筑深度(5)。→ playtest 定。
2. **答错惩罚多重?** 当前只清连击;要不要扣血/给对手能量?影响容错流价值。
3. **抽取 3 次够不够深?** SAP 说少件好平衡,但成瘾要够多决策。→ 测"再来一局率"反推。
4. **自适应难度 vs 纯对称竞技**:排位是否完全对称?学力差距大的怎么匹配?
5. **团队跨玩家 ×倍率 上限**怎么设,才既有协同爽感又不炸穿?
6. **PVE/闯关模式**要不要?(留存补充 + 新手教学 + 单人也能玩)
